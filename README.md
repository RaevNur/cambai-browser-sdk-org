# Camb.ai Browser SDK

The official JavaScript SDK for [Camb.ai](https://camb.ai), designed for use in browser environments (React, Vue, Vanilla JS, etc.).

## Features

- üöÄ **Browser Compatible**: No Node.js dependencies (fs, path, etc.).
- üîä **Full API Support**: Text-to-Speech (TTS), Voice Cloning, Dubbing.
- üîí **Type-Safe**: Written in TypeScript with full type definitions.
- üì¶ **Lightweight**: Optimized for bundling with Vite, Webpack, or Rollup.

## Installation

```bash
npm install @camb-ai/browser-sdk
```

## Usage

### Basic Text-to-Speech (TTS)

```javascript
import { CambClient } from '@camb-ai/browser-sdk';

const client = new CambClient({
  apiKey: "YOUR_API_KEY" // ‚ö†Ô∏è Note: In production, use proxy instead of exposing key
});

async function playGreeting() {
  const response = await client.textToSpeech.tts({
    text: "Hello, this is a voice generated in the browser!",
    language_id: 1, // English
    voice_id: 20303, // Standard Voice
    gender: 1,
    age: 1
    // debug: true (optional)
  });

  // The response contains a stream, but we likely want a blob for the browser
  // (Assuming your backend proxy handles the implementation details, 
  // or you use our upcoming polling/websocket features)
  console.log("TTS task initiated", response);
}
```

### Dubbing Example

```javascript
import { CambClient } from '@camb-ai/browser-sdk';

const client = new CambClient({ apiKey: "YOUR_API_KEY" });

async function dubVideo(file) {
  // 'file' is a browser File object (e.g. from <input type="file">)
  
  const response = await client.dub.dubVideo({
     video_file: file,
     source_language: 1, // English
     target_language: 2  // Spanish
  });
  
  console.log("Dubbing ID:", response.task_id);
}
```

## ‚ö†Ô∏è Important Security Note

**Never expose your API Key in frontend code that is publicly accessible.**

For production applications, we recommend:
1.  Sending requests to your own backend.
2.  Your backend attaches the `CAMB_API_KEY` and forwards the request to Camb.ai.
3.  Or use a short-lived token generated by your backend (feature coming soon).
